# 开发环境与发布规范

> 本文档描述 NcatBot 项目的开发环境配置、CI/CD 流水线、PyPI 发布流程及项目基础设施约定。

## 1. Python 版本要求

- **最低版本**：Python 3.12
- **测试矩阵**：3.12、3.13
- **约束声明位置**：`pyproject.toml` → `requires-python = ">=3.12"`

本项目不再支持 Python 3.11 及以下版本。相关的兼容性依赖（如 `tomli`）已移除，标准库 `tomllib` 可直接使用。

## 2. 环境管理

### 2.1 工具链

| 工具 | 用途 |
|------|------|
| [uv](https://docs.astral.sh/uv/) | 虚拟环境管理、依赖解析、包构建 |
| [ruff](https://docs.astral.sh/ruff/) | 代码检查（lint）+ 格式化（替代 black/isort/flake8） |
| [pre-commit](https://pre-commit.com/) | Git 提交钩子，自动执行 lint 和格式检查 |
| [tox](https://tox.wiki/) + tox-uv | 多版本测试 |
| [pytest](https://docs.pytest.org/) | 单元测试框架 |

### 2.2 初始化开发环境

```bash
# 克隆仓库
git clone https://github.com/<your-username>/ncatbot.git
cd ncatbot

# 安装开发依赖（自动创建 .venv）
uv sync --extra dev

# 激活虚拟环境
source .venv/bin/activate      # Linux/macOS
.venv\Scripts\activate.ps1     # Windows PowerShell

# 安装 pre-commit 钩子（仅需一次）
uv run pre-commit install
```

### 2.3 依赖管理

依赖声明在 `pyproject.toml` 的 `[project.dependencies]`，锁定文件为 `uv.lock`（已纳入版本控制）。

```bash
# 添加/修改依赖后重新锁定
uv lock

# 升级所有依赖到最新版本
uv lock --upgrade

# 同步本地环境
uv sync --extra dev
```

> **注意**：`pyproject.toml` 和 `uv.lock` 必须一起提交，以保证 CI 和其他开发者的环境可复现。

### 2.4 依赖分组

| Extra | 说明 | 安装方式 |
|-------|------|----------|
| `test` | pytest 及相关插件 | `uv sync --extra test` |
| `dev` | 完整开发工具链（包含 test） | `uv sync --extra dev` |

## 3. 代码质量

### 3.1 Lint & Format

本项目统一使用 [Ruff](https://docs.astral.sh/ruff/) 进行代码检查和格式化：

```bash
# 检查
uv run ruff check .

# 自动修复
uv run ruff check --fix .

# 格式化
uv run ruff format .
```

> 不再使用 `black`、`isort`、`flake8`。Ruff 已覆盖其全部功能。

### 3.2 pre-commit 钩子

`.pre-commit-config.yaml` 定义了以下自动检查：

| 钩子 | 作用 |
|------|------|
| `trailing-whitespace` | 移除行尾空格 |
| `end-of-file-fixer` | 确保文件以换行符结尾 |
| `check-yaml` / `check-toml` / `check-ast` | 语法校验 |
| `check-merge-conflict` | 检测未解决的合并冲突 |
| `detect-private-key` | 检测意外提交的私钥 |
| `mixed-line-ending` | 统一为 LF 换行符 |
| `ruff` | 代码检查 + 自动修复 |
| `ruff-format` | 代码格式化 |
| `codespell` | 拼写检查 |

### 3.3 测试

```bash
# 运行测试
uv run pytest --no-cov

# 运行带覆盖率的测试
uv run pytest

# 多版本测试（3.12 + 3.13）
uv run tox
```

## 4. CI/CD 流水线

### 4.1 测试流水线（`.github/workflows/pytest.yml`）

**触发条件**：
- push 到 `main` 分支
- 任何分支的 Pull Request

**执行步骤**：
1. **lint**：运行 `ruff check` 和 `ruff format --check`
2. **test**：在 Python 3.12 和 3.13 上运行 `pytest --no-cov`（lint 通过后）

### 4.2 发布流水线（`.github/workflows/pypi-publish.yml`）

**触发条件**：
- push 以 `v` 开头的 tag（如 `v5.0.0rc7`、`v5.0.0`）
- 手动触发（`workflow_dispatch`）

**执行步骤**：
1. `uv build` — 构建 sdist 和 wheel
2. `twine check dist/*` — 验证包元数据
3. `pypa/gh-action-pypi-publish` — 发布到 PyPI

> **重要**：不再因 `pyproject.toml` 文件变化而自动发布。只有创建 Git tag 才会触发发布。

## 5. 版本号与发布流程

### 5.1 版本号规范

遵循 [PEP 440](https://peps.python.org/pep-0440/)：

| 阶段 | 格式 | 示例 |
|------|------|------|
| 开发版 | `X.Y.ZdevN` | `5.0.0dev1` |
| Alpha | `X.Y.ZaN` | `5.0.0a1` |
| Beta | `X.Y.ZbN` | `5.0.0b1` |
| RC | `X.Y.ZrcN` | `5.0.0rc7` |
| 正式版 | `X.Y.Z` | `5.0.0` |
| 补丁版 | `X.Y.Z` | `5.0.1` |

> **注意**：不要使用 `5.0.0-rc7` 这种带连字符的格式，PEP 440 不合规。

版本号在 `pyproject.toml` 的 `[project].version` 字段中手动管理（不使用 `setuptools_scm`）。

### 5.2 发布步骤

```bash
# 1. 更新 pyproject.toml 中的 version 字段
# 2. 提交版本变更
git add pyproject.toml uv.lock
git commit -m "release: v5.0.0"

# 3. 创建 tag
git tag v5.0.0

# 4. 推送 tag（自动触发 CI 发布）
git push origin main --tags
```

### 5.3 本地验证构建

```bash
uv build
uvx twine check dist/*
```

## 6. 项目基础设施文件

| 文件 | 说明 |
|------|------|
| `pyproject.toml` | 包元数据、依赖、工具配置（pytest/coverage/ruff/codespell） |
| `uv.lock` | 依赖锁定文件（已纳入版本控制） |
| `.python-version` | 声明项目默认 Python 版本（3.12），uv/pyenv 会自动识别 |
| `.editorconfig` | 编辑器统一配置（UTF-8、LF、4 空格缩进） |
| `.gitattributes` | Git 换行符规范化（统一 LF） |
| `.gitignore` | Git 忽略规则 |
| `.pre-commit-config.yaml` | pre-commit 钩子配置 |
| `tox.ini` | 多版本测试配置 |
| `CONTRIBUTING.md` | 贡献指南 |
| `LICENSE` | 许可证 |

### 6.1 换行符（Line Endings）

本项目统一使用 **LF**（`\n`）换行符，通过三层机制保证：

1. `.gitattributes`：`* text=auto eol=lf` — Git checkout 时自动转换
2. `.editorconfig`：`end_of_line = lf` — 编辑器保存时使用 LF
3. `.pre-commit-config.yaml`：`mixed-line-ending --fix=lf` — 提交时自动修正

### 6.2 .gitignore 组织

`.gitignore` 只包含本项目实际需要忽略的内容，按类别分组：
- Python 编译产物（`__pycache__/`、`*.pyc`）
- 构建产物（`build/`、`dist/`、`*.egg-info/`）
- 虚拟环境（`.venv/`）
- 测试/覆盖率（`htmlcov/`、`.tox/`、`.coverage`）
- IDE 配置（`.vscode/`、`.idea/`）
- 运行时数据（`logs/`、`/data`、`/plugins`、`config.yaml`）
- 开发目录（`dev/`）

> 不包含与项目无关的框架条目（Django、Flask、Scrapy 等）。

## 7. 目录约定

```
NcatBot/
├── ncatbot/           # 核心源码
├── tests/             # 测试代码
├── docs/              # 开发者文档
├── examples/          # 示例代码
├── data/              # 插件测试数据
├── dev/               # 本地开发脚本（.gitignore 排除，不入版本控制）
├── ncatbot_legacy/    # 旧版代码（暂时保留）
├── test_legacy/       # 旧版测试（暂时保留）
├── pyproject.toml     # 包配置
├── uv.lock            # 依赖锁定
└── tox.ini            # 多版本测试
```

- `dev/` 目录用于存放本地开发脚本和配置，不纳入版本控制
- `config.yaml` 为运行时配置，不纳入版本控制
- `ncatbot_legacy/` 和 `test_legacy/` 为旧版代码，后续版本将清理
