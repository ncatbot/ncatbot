# 代码库导航（协调器）

本技能串联 **doc-nav**（文档导航）和 **code-nav**（代码定位）两个子技能，完成从问题到代码的完整定位流程。

## 完整工作流

```text
1. 分类问题 → 2. 查阅文档（→ doc-nav）→ 3. 定位代码（→ code-nav，仅在必要时）
```

---

## 路由决策

| 用户意图 | 委托给 | 说明 |
|---------|--------|------|
| 理解功能、查 API 签名、了解预期行为 | **doc-nav** | 从文档获取答案，不需要读代码 |
| 找实现文件、追踪调用链、定位 bug 行号 | **code-nav** | 需要精准定位到代码位置 |
| 不确定问题在哪 / 需要完整排查 | **doc-nav → code-nav** | 先通过文档理解，再定位代码 |

---

## 典型流程

### 场景 1：用户描述了一个问题症状

1. 使用 **doc-nav** 的症状速查表，确定涉及的模块和应读的文档
2. 阅读文档，理解预期行为
3. 如需确认实际实现或定位 bug 行号，使用 **code-nav** 锁定具体文件

### 场景 2：用户想了解某个功能怎么实现

1. 使用 **doc-nav** 查阅对应的 guide 和 reference 文档
2. 如文档不足以回答，使用 **code-nav** 找到实现代码

### 场景 3：用户直接要求找某个类/函数

1. 跳过 **doc-nav**，直接使用 **code-nav** 的模块速查表定位

---

## 核心原则

- **文档优先**：先读文档再读代码，通过文档理解"应该怎样"
- **按需深入**：只有文档无法回答时才读源代码
- **搜索定位**：用 Explore 子代理搜索关键类名，不遍历模块
